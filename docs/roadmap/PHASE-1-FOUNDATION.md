- Status: Complete on `chore/open-source-foundation`

Establish the repository policies, contribution process, documentation hierarchy, review templates, and initial governance before implementation begins.

- Contribution guidelines
- Contributor Code of Conduct
- Security policy
- Governance model
- AI-assisted development policy
- Changelog
- Issue forms and pull request template

## Deliberately deferred

The following require an accepted implementation stack or architecture and are not guessed in Phase 1:

- language-specific `.gitignore` rules;
- build and test workflows;
- dependency update configuration;
- release automation;
- package publishing;
- runtime-specific security scanning;
- source-code directory layout.

These items must be added when their technical requirements are known.
