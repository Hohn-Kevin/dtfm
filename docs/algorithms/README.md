# Algorithms

This directory documents algorithms and heuristics used by DTFM.

Each algorithm document should cover:

- the problem being solved;
- inputs and outputs;
- assumptions and invariants;
- deterministic and non-deterministic behavior;
- complexity and resource limits;
- failure conditions;
- evaluation criteria;
- security and privacy implications;
- known limitations.

An implementation is not considered authoritative when it conflicts with an accepted specification.
