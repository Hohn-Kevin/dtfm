# Architecture Decision Records

Architecture Decision Records preserve significant technical decisions and their context.

## Naming

Use sequential names such as:

`0001-select-primary-implementation-language.md`

## Status

Use one of:

- Proposed
- Accepted
- Rejected
- Deprecated
- Superseded

Do not delete accepted decisions. Mark them as superseded and link the replacement ADR.

## Template

Copy `0000-template.md` when proposing a new decision.
